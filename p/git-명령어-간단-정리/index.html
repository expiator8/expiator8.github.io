Git 명령어 간단 정리

Git 저장소로 초기화 

1

git init

원격 저장소 연결 

1

git remote add origin [원격 저장소 url]

-원격 저장소를 현재 로컬 Git 저장소에 연결

변경된 파일 추가 

1

git add [파일명]

-현재 디렉토리의 변경된 파일을 스테이지에 추가

e.g.

1
2

git add . > 모든 변경된 파일 추가
git add models/users.py > users.py 파일만 추가

추가된 파일 커밋 

1

git commit -m "message"

-스테이지에 추가된 파일들을 커밋해 로컬 Git 저장소에 저장

커밋 반영 

1

git push origin main

-로컬 저장소 main 브랜치의 커밋을 원격 저장소의 main 브랜치로 반영

1

git push -u origin main

--u옵션: –set-upstream의 축약형
--u옵션을 사용하고 브랜치를 설정한 이후부터는 원격 브랜치를 명시하지 않아도 자동 추적
i.e. git push, git pull만 해도 origin main 브랜치로 명령 동작

1

git push -f origin main

--f옵션: 강제 반영. 로컬 저장소의 커밋을 원격 저장소의 커밋에 덮어쓰기.
-권장xx 쓴다면 주의 또 주의. 코드 다날라간다

브랜치

생성 

1

git branch main

-main 브랜치 생성

변경 

1

git branch -M main

-현재 브랜치의 이름을 main으로 변경
-이미 main 브랜치가 존재한다면 덮어씀

1

git branch -m master main

-현재 브랜치의 이름을 main으로 변경
-이미 main 브랜치가 존재한다면 오류 발생

삭제 

1

git branch -d main

-main 브랜치 삭제

전환 

1

git switch main

-이미 있는 main 브랜치로 전환

1

git switch -c main

--c옵션: main 브랜치 생성 및 전환
** git checkout 명령어가 브랜치 전환, 파일 복원 두가지 기능이 있는데
switch는 브랜지 전환, restore은 파일 복원으로 보다 직관적으로 분리된 명령어임

최신상태 가져오기 

1

git fetch --all

-모든 원격 저장소에서 최신 브랜치, 커밋, 태그 정보를 가져옴
-로컬 브랜치에는 영향을 미치지 않고, 원격 저장소의 변경 사항을 업데이트만 한다

클론 

1

git clone [원격 저장소 url]

-원격 저장소를 로컬로 복제. 원격 저장소의 모든 브랜치, 커밋, 태그를 포함해 새로운 디렉토리로 복사

리셋 

1

git reset

현재 브랜치의 HEAD를 지정한 커밋 등으로 이동 HEAD1는 현재 HEAD 위치에서 1개. 즉 최근 커밋 1개 HEAD2는 HEAD 위치에서부터 커밋 2개

1

git reset --soft HEAD~1

--soft: 커밋을 되돌리되, 변경 사항은 스테이징 영역에 유지

1

git reset --mixed HEAD~1

--mixed (default): 커밋을 되돌리고, 변경 사항을 작업 디렉토리에 유지. 스테이징 영역은 초기화

1

git reset --hard HEAD~1

--hard: 커밋을 되돌리고, 변경 사항을 작업 디렉토리와 스테이징 영역에서 모두 삭제.
주의: 되돌린 변경 사항은 복구 불가!

1

git reset --hard origin/main

원격 저장소의 main 브랜치의 최근 상태로 리셋

임시저장 

1

git stash

임시저장하기

1

git stash list

임시저장 목록 확인

1

git stash apply

가장 최근 stash 불러오기

1

git stash apply [stash 이름]

해당 이름의 stash 불러오기

1

git stash drop

가장 최근 stash 삭제

1

git stash drop [stash 이름]

해당 이름의 stash 삭제

1

git stash pop

apply + drop
